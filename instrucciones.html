Segundo Previo - Computacion en la Nube

Cloud Store Lab - Google Cloud + FastAPI

Modalidad

Trabajo en parejas.

Al menos uno de los integrantes debe tener acceso funcional a Google Cloud y permisos suficientes para desplegar recursos.

Objetivo

El objetivo de esta actividad NO es evaluar FastAPI como framework.

El objetivo es evaluar su capacidad para:

  • Configurar servicios cloud reales.
  • Resolver problemas de conectividad.
  • Manejar permisos e IAM.
  • Trabajar con variables de entorno.
  • Integrar multiples servicios administrados de Google Cloud.
  • Desplegar una aplicacion funcional en la nube.

Descripcion General

Se les entregara un repositorio base minimo con:

  • main.py
  • requirements.txt
  • .env.example
  • README.md

El repositorio tendra endpoints vacios (pass) y comentarios guia.

Ustedes deberan completar la integracion cloud y lograr que la aplicacion funcione completamente desplegada en Google Cloud.

Arquitectura Esperada

Servicio Uso
App Engine Standard Hosting de la API
Cloud SQL (PostgreSQL) Persistencia relacional
Cloud Storage Almacenamiento de imagenes
Firestore Eventos/auditoria
Variables de entorno Configuracion

Requisitos Tecnicos

1. Ejecutar localmente

Deben demostrar que la aplicacion puede correr localmente:

  • Directamente desde el host.
  • O usando Docker (opcional).

2. Desplegar en Google Cloud

La aplicacion debe ejecutarse en:

  • Google App Engine Standard

3. Integrar Cloud SQL

Deben:

  • Crear instancia PostgreSQL.
  • Crear base de datos.
  • Conectarse usando psycopg2.
  • Crear y consultar informacion.

4. Integrar Cloud Storage

Deben:

  • Crear bucket.
  • Subir imagenes.
  • Resolver permisos correctamente.
  • Evitar errores 403 Forbidden.

5. Integrar Firestore

Deben:

  • Registrar eventos/auditoria.
  • Consultar eventos.

Endpoints Esperados

  • GET /health - Health check.
  • POST /products - Crear producto.
  • GET /products - Listar productos.
  • POST /products/{product_id}/image - Subir imagen de producto.
  • POST /products/{product_id}/comments - Crear comentario.
  • GET /audit/events - Consultar eventos.

Sustentacion

La sustentacion sera:

  • En vivo.
  • En clase.
  • Individual.
  • Maximo 8 minutos por grupo.
IMPORTANTE:

Deben venir preparados. No habra tiempo para:

  • Instalar dependencias.
  • Crear buckets.
  • Arreglar permisos.
  • Buscar documentacion.
  • Depurar despliegues.

La aplicacion debe llegar funcionando.

Pruebas HTTP

Durante la sustentacion deberan demostrar el funcionamiento de los endpoints usando:

  • Postman
  • Insomnia
  • Bruno
  • o Cartero

Pueden usar cualquier cliente REST.

IMPORTANTE:

Deben ser capaces de:

  • Subir imagenes.
  • Enviar JSON.
  • Probar endpoints en remoto.
  • Demostrar respuestas funcionales.

La resolucion de como enviar archivos multipart/form-data queda completamente a cargo de ustedes.

Lo que se espera de ustedes

Se espera que:

  • Investiguen.
  • Lean documentacion.
  • Solucionen errores.
  • Entiendan IAM.
  • Entiendan variables de entorno.

Recomendaciones

  • No dejar permisos para ultima hora.
  • Probar el despliegue varios dias antes.
  • Validar que App Engine realmente puede conectarse a Cloud SQL.
  • Validar que Firestore realmente registra eventos.

  • Practicar la sustentacion antes de la clase.
  • Tener claro que hizo cada integrante.
